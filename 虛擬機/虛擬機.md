# 1.3 Process Virtual Machine
> Process-level VM 提供 virtual ABI environment
- multi-programming
	- simultaneously support multiple user processes
- emulator and dynamic binary transistors
	- emulation
		- interpretation : most straightforward emulation
			- target ISA fetches, decodes, emulate the execution of individual source instructions
		- problem : relatively <u>slow</u> process, so the better solution is  binary translation
	- dynamic binary translators
		- `block` of source instructions are converted to target instructions that perform equivalent functions
		- 雖然說 translation 的 process 可能會有 relatively high overhead, 但是可以 cached and repeatedly executed
	- comparison
		- emulation
			- relatively slow start up, consume significant time when emulated
		- binary translation
			- high initial overhead, but fast repeated execution
	- 所以，有些 VM 有 stage emulation combined with `profiling
		- profiling 剛開始 determine <u>frequently executed</u> instruction sequences，然後用 binary transition
- same  ISA binary translation
	- 如果 guest 跟 host 的 instruction sets 是一樣的
	- 包括 stage optimization & software caching
- High Level Language Virtual machine : platform independence
	![[Pasted image 20260414193126.png]]
	- cross-platform `portability`
	- code generator (我不知道為啥要說這個)
		- generate binary containing machine code, for specific ISA and OS
	- advantage of HLL VM
		- software easily portable, once VM is implemented on target platform
		- 雖然說 VM implementation 會需要一些努力，但是相較於 developing a compiler for each platform & recompiling every application when it is ported
		- simpler than developing a conventional emulating process VM for typical real-word ISA
# 1.4 System Virtual Machine
1. complete system environment provide multiple user coexist
2. history
	1. 以前的 mainframe computer very large & expensive
		1. shared large number of users
	2. 隨著 hardware 變便宜，轉向 destop，VM faded
	3. today, VM 重回 popular
		1. security (isolation) ![[Pasted image 20260414195345.png]]
			1. partition major system that run concurrently on same hardware platform 
			2. software running on guest system is isolated from hardware running on other guest systems
3. implementation of system virtual machine
4. whole system VMs : Emulation (所以 這到底是什麼) ![[Pasted image 20260414200305.png]]
	1. system software (guest & host) and application software
	2. def : support on host system that runs a different ISA and OS
	3. convert guest system ISA operations, equivalent to OS calls made to OS host
5. co-designed Virtual machine : hardware optimization
	1. enable innovate ISA and/or hardware implementations for improve performance, power efficiency
	2. complex instructions decomposed into simpler, Risc like instruction
	3. goal : provide VM look exactly like a naive hardware platform
		1. VM use a region of memory that is not visible to any application or system software
			1. binary translator convert guest instructions into naive ISA instructions and cached the translated instructions in a region of concealed(隱藏的) memory
	4. for performance, translation is coupled with code optimization
